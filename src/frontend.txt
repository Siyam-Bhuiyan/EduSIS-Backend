here’s your **overall Frontend → Backend Mapping Quick Guide** to wire every screen fast. keep your `Authorization: Bearer <token>` header on.

---

# Base

* Base URL: `http://localhost:5000`
* Auth:

  * Register: `POST /api/auth/register`
  * Login: `POST /api/auth/login` → `{ token }`
  * Me: `GET /api/auth/me`

---

# Global / Profile

* `Profile.jsx`

  * Get me: `GET /api/profile/me`
  * Update me: `PATCH /api/profile/me`
  * Change password: `PATCH /api/profile/me/password`
* `UpcomingEventsPanel.jsx` (student/teacher dashboards)

  * My events (classes + assignment due): `GET /api/calendar/me?from=:ISO&to=:ISO`

---

# Admin

* `AdminDashboard.jsx`

  * Stats: `GET /api/admin/dashboard/stats`
* `Students.jsx` (admin CRUD + self read)

  * List: `GET /api/students?page&limit&search`
  * Create: `POST /api/students` (admin)
  * Get: `GET /api/students/:id` (self/admin)
  * Update: `PATCH /api/students/:id` (self/admin)
  * Delete: `DELETE /api/students/:id` (admin)
  * Current student profile: `GET /api/students/me/profile`
* `Teachers.jsx` (admin CRUD + self read)

  * List: `GET /api/teachers?page&limit&search`
  * Create: `POST /api/teachers` (admin)
  * Get: `GET /api/teachers/:id` (self/admin)
  * Update: `PATCH /api/teachers/:id` (self/admin)
  * Delete: `DELETE /api/teachers/:id` (admin)
  * Current teacher profile: `GET /api/teachers/me/profile`
* `Department` admin (if you have a view)

  * List: `GET /api/departments`
  * Create: `POST /api/departments` (admin)
  * Get: `GET /api/departments/:id`
  * Update: `PATCH /api/departments/:id` (admin)
  * Delete: `DELETE /api/departments/:id` (admin)

---

# Courses & Members

* `Courses.jsx`

  * List: `GET /api/courses?page&limit&search&department_id`
* `CourseCard.jsx`

  * Use fields from list: `id, title, code, credit, department_name`
* `CourseDetail.jsx` (student view) / `TeacherCourseDetail.jsx` (teacher view)

  * Overview bundle: `GET /api/courses/:id/overview`
    (returns: course, teachers\[], enrolled\_count, my\_membership, announcements\[], upcoming\_classes\[], recent\_assignments\[], recent\_messages\[])
  * Raw course (if needed): `GET /api/courses/:id`
* `TeacherAssignment.jsx` (assign teachers to course)

  * Get current teachers: `GET /api/courses/:id/teachers`
  * Set/replace: `POST /api/courses/:id/teachers` (admin)

---

# Enrollment (students)

* `StudentEnrollment.jsx`

  * Enroll: `POST /api/enrollments/me` `{ "course_id": :id }`
  * Drop: `DELETE /api/enrollments/me/:course_id`
  * My courses: `GET /api/enrollments/me`
* Admin enrollments (if needed in admin UI)

  * By course: `GET /api/enrollments/course/:course_id` (admin)

---

# Assignments, Submissions, Grades

* `Assignments.jsx` (list + create/edit + submit)

  * List by course: `GET /api/assignments?course_id=:id&page&limit`
  * Get one (student also sees own submission/grade): `GET /api/assignments/:id`
  * Create: `POST /api/assignments` (teacher/admin)
  * Update: `PATCH /api/assignments/:id` (teacher/admin)
  * Delete: `DELETE /api/assignments/:id` (teacher/admin)
  * Submit: `POST /api/assignments/:id/submit` (student) `{ file_url?, text_answer? }`
  * Teacher view submissions: `GET /api/assignments/:id/submissions` (teacher/admin)
  * Grade: `POST /api/assignments/:id/grade/:student_id` (teacher/admin) `{ score, feedback? }`
* `AssignGrades.jsx`

  * Use the submissions + grade endpoints above
* `Results.jsx` (student)

  * My graded results: `GET /api/assignments/me/results/list`

---

# Announcements

* `Announcements.jsx`

  * List: `GET /api/announcements?course_id=:id&page&limit`
    (visibility auto-filtered by role)
  * Get one: `GET /api/announcements/:id`
  * Create: `POST /api/announcements` (teacher/admin)
  * Update: `PATCH /api/announcements/:id` (owner/admin)
  * Delete: `DELETE /api/announcements/:id` (owner/admin)

---

# Online Classes (Jitsi)

* `OnlineClasses.jsx`

  * List by course/upcoming: `GET /api/classes?course_id=:id&upcoming=true&page&limit`
  * Create: `POST /api/classes` (teacher/admin)
  * Update/Delete: `PATCH /api/classes/:id`, `DELETE /api/classes/:id` (owner/admin)
  * Role feeds:

    * Student upcoming: `GET /api/classes/me/student/upcoming`
    * Teacher upcoming: `GET /api/classes/me/teacher/upcoming`
* `JitsiMeetingScreen.jsx`

  * Join (get URL): `GET /api/classes/:id/join` → `{ meeting_url }`

---

# Messages (course chat)

* `Messages.jsx`

  * List: `GET /api/courses/:course_id/messages?limit=30&after_id?&before_id?`
    (member-only: enrolled student / assigned teacher / admin)
  * Send: `POST /api/courses/:course_id/messages` `{ body | attachment_url }`
  * Edit: `PATCH /api/courses/:course_id/messages/:message_id`
  * Delete: `DELETE /api/courses/:course_id/messages/:message_id`
  * Read receipt (optional): `POST /api/courses/:course_id/messages/read` `{ message_id }`
  * Readers (optional): `GET /api/courses/:course_id/messages/:message_id/readers`

---

# Dashboards

* `StudentDashboard.jsx`

  * My events: `GET /api/calendar/me` (limit window by query)
  * My courses: `GET /api/enrollments/me`
  * Latest announcements (global): `GET /api/announcements?page=1&limit=5`
* `TeacherDashboard.jsx`

  * Teaching courses: `GET /api/courses/me/teaching/list`
  * Upcoming classes: `GET /api/classes/me/teacher/upcoming`
  * Recent announcements you created (optional filter client-side): `GET /api/announcements?page=1&limit=5`

---

# Common patterns & tips

* **Auth header** everywhere after login: `Authorization: Bearer <token>`.
* **RBAC**:

  * Admin: full CRUD on everything.
  * Teacher: manages own assignments/announcements/classes; views own courses.
  * Student: enroll/submit/view own stuff; read messages in enrolled courses.
* **Pagination**: `page`, `limit` on list endpoints; chats use `after_id`/`before_id`.
* **Dates** pass/receive as ISO strings (`YYYY-MM-DDTHH:mm:ssZ`).
* **Conflicts** (409): duplicate email, course code, department code, etc.
* **Course membership required** for messages endpoints.
