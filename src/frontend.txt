how this maps to your frontend right now

Login.jsx / Register.jsx → use /api/auth/login and /api/auth/register to authenticate; store the returned token in local storage (or httpOnly cookie if you prefer later).

Any page that needs the current user (e.g., StudentDashboard.jsx, TeacherDashboard.jsx, AdminDashboard.jsx, Profile.jsx) → call /api/auth/me with the Authorization: Bearer <token> header.



how this maps to your frontend

Students.jsx (admin): use list /api/students, create /api/students, edit /api/students/:id, delete /api/students/:id.

StudentDashboard.jsx / Profile.jsx: use GET /api/students/me/profile to show the student’s profile; for edits, call PATCH /api/students/:id (their own id from /api/auth/me).


Frontend mapping

Teachers.jsx (admin directory): use list, create, edit, delete above.

TeacherDashboard.jsx / Profile.jsx: use GET /api/teachers/me/profile; for edits call PATCH /api/teachers/:id (id from /api/auth/me).


Frontend mapping quick guide

Courses.jsx → GET /api/courses (+ search/department filters).

CourseCard.jsx → uses list data fields title, code, credit, department_name.

CourseDetail.jsx → GET /api/courses/:id (includes teachers[], enrolled_count).

TeacherAssignment.jsx (admin) → GET /api/courses/:id/teachers, POST /api/courses/:id/teachers.

StudentEnrollment.jsx → POST /api/enrollments/me (Enroll), DELETE /api/enrollments/me/:course_id (Drop), GET /api/enrollments/me (list my courses).

TeacherCourseDetail.jsx → use GET /api/courses/me/teaching/list + /api/courses/:id.


Frontend mapping quick guide

Assignments.jsx (student view): GET /api/assignments?course_id=:id, GET /api/assignments/:id

Assignments.jsx (submission): POST /api/assignments/:id/submit

Assignments.jsx (teacher create/edit): POST /api/assignments, PATCH /api/assignments/:id, DELETE /api/assignments/:id

AssignGrades.jsx: GET /api/assignments/:id/submissions, POST /api/assignments/:id/grade/:student_id

Results.jsx: GET /api/assignments/me/results/list
